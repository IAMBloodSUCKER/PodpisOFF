# Telegram-бот ПодписOFF

Официальный бот для сервисных уведомлений: напоминания о списаниях, ответы поддержки, смена тарифа.

Аватар для BotFather: [`docs/assets/telegram-bot-avatar.png`](../docs/assets/telegram-bot-avatar.png) (512×512, `/setuserpic` в BotFather).

## 1. Создание в BotFather

Откройте [@BotFather](https://t.me/BotFather) в Telegram.

### Имя и username

| Поле | RU (ПодписOFF) | EN (SubOFF) |
|------|----------------|-------------|
| **Имя бота (Name)** | `PodpisOFFBot` | `SubOFFBot` |
| **Username** | `PodpisOFF_bot` | `SubOFF_bot` |

> Username должен быть уникальным и заканчиваться на `bot`.

### Команды

```

```

В BotFather: `/setcommands` → выберите бота → вставьте список выше.

> Управление — **только кнопками** в одном сообщении. Команды `/subs`, `/summary` и т.д. больше не нужны: бот обновляет то же сообщение при нажатии кнопок.

## 2. Webhook (продакшен)

Бот получает сообщения через webhook на бэкенд:

```
POST https://api.ваш-домен.ru/api/telegram/webhook
```

При старте приложения webhook регистрируется автоматически, если заданы `TELEGRAM_BOT_TOKEN` и `TELEGRAM_WEBHOOK_PUBLIC_URL`.

Ручная регистрация:

```bash
curl "https://api.telegram.org/bot<TOKEN>/setWebhook" \
  -d "url=https://api.ваш-домен.ru/api/telegram/webhook" \
  -d "secret_token=<TELEGRAM_WEBHOOK_SECRET>"
```

### Локальная разработка

> **Важно:** бэкенд должен достучаться до `api.telegram.org` (исходящий HTTPS). В некоторых сетях Telegram заблокирован — тогда `/start` и уведомления **не заработают** локально. Имеет смысл проверять интеграцию на **staging/проде**, а не на домашнем ПК без VPN.

Webhook требует публичный HTTPS. Варианты:

- [ngrok](https://ngrok.com/) → `TELEGRAM_WEBHOOK_PUBLIC_URL=https://xxxx.ngrok.io` (всё равно нужен доступ к `api.telegram.org`)
- Деплой на staging с реальным доменом — **рекомендуемый** способ проверки
- Без `TELEGRAM_WEBHOOK_PUBLIC_URL` бэкенд сам опрашивает Telegram (polling) — подходит для dev, если API доступен

## 3. Сценарий для пользователя

### В настройках на сайте

1. **Настройки** → **Уведомления на почту и в Telegram**
2. Пользователь читает, что бот присылает
3. Ставит галочку **«Согласен получать уведомления от @PodpisOFFBot…»** — это принятие условий
4. Нажимает **«Подключить Telegram»**
5. Видит **QR-код** и **ссылку** вида `https://t.me/PodpisOFF_bot?start=<токен>`
6. На экране: «Ожидаем подключения…»

### В Telegram

1. Пользователь сканирует QR или открывает ссылку
2. Telegram открывает чат с ботом
3. Пользователь нажимает **«Запустить» / Start**
   - Telegram передаёт боту команду `/start <токен>`
4. Бэкенд по webhook:
   - проверяет одноразовый токен (15 минут)
   - сохраняет `chat_id` в аккаунт пользователя
   - включает `telegram_notifications_enabled`
5. Бот показывает **одно сообщение с кнопками** (меню). Дальше пользователь нажимает кнопки — то же сообщение обновляется.

### На сайте

- Статус меняется на **«Telegram подключён»**
- Уведомления включены автоматически

### Отключение

- **На сайте:** «Отключить Telegram»
- **В боте:** кнопка «Отключить уведомления» в меню (сообщение обновится)

## 4. Что умеет бот

### Меню с кнопками (одно сообщение)

| Кнопка | Аналог на сайте | Тариф |
|--------|-----------------|-------|
| Подписки | Список подписок | Все |
| Ближайшие | Ближайшие списания (31 день) | Все |
| Сводка | Дашборд — сводка расходов за месяц | Все (навигация по месяцам — **PRO**) |
| Аккаунт | Настройки — аккаунт и тариф | Все |
| Аналитика | Расширенная аналитика дашборда | **PRO** |
| Экспорт Excel | Выгрузка подписок | **PRO** |
| Отключить уведомления | Отключить Telegram | Все |

### PRO в боте

На тарифе **PRO** в меню появляются дополнительные кнопки:

- **Аналитика** — тренд расходов за 6 месяцев, топ подписок и разбивка по категориям
- **Экспорт Excel** — файл `podpisoff-subscriptions.xlsx` прямо в чат
- **Сводка** — кнопки «← Месяц» / «Месяц →» для просмотра прошлых месяцев

На бесплатном тарифе при нажатии PRO-функций показывается экран с описанием и кнопкой «Оформить PRO» (ссылка на `/settings#plans`).

Повторный **Start** обновляет то же меню (или заменяет старое). Произвольный текст в чате бот **игнорирует**.

Редактирование подписок и оплата PRO — на сайте.

### Push-уведомления

Отдельные сообщения в Telegram (не меню), если канал включён:

- подписка **добавлена**, **изменена** или **удалена** на сайте
- напоминания о списаниях
- ответы на отзыв / поддержку
- сообщения администратора
- смена тарифа

После каждого push-уведомления бот **пересылает меню вниз чата** (старое сообщение с кнопками удаляется). Кнопки всегда остаются под последним уведомлением — листать вверх не нужно.

Меню с кнопками — одно сообщение, обновляется при нажатии.

## 5. Безопасность

- Ссылка подключения одноразовая, срок 15 минут
- Webhook защищён `TELEGRAM_WEBHOOK_SECRET` (заголовок `X-Telegram-Bot-Api-Secret-Token`)
- Токен бота только на сервере, не в фронтенде
- В логах токен **не выводится** — при ошибках API URL редактируется
- Пользователь не вводит chat id вручную
